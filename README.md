# Лабораторна робота 3 — прогрес StudyManager
### 1) Перехід на MVVM
- Логіку взаємодії з UI перенесено у **ViewModels** (вибір елемента, команди, перехід між екранами).
- У `.xaml.cs` файлах залишено тільки `InitializeComponent()` та задання `DataContext`.

### 2) Багатошарова архітектура
- **Storage (DB Models)**: моделі збереження даних (SubjectRecord, LessonRecord, enums).
- **Repositories**: робота зі сховищем та отримання DB Models через інтерфейси (`IStudyStore`, `IStudyRepository`).
- **Services**: підготовка даних для UI через DTO та сервіси (`IStudyService`), конвертація DB Models → DTO.
- **UI (WPF)**: використовує тільки Services та DTO, не звертається напряму до Repositories/DB Models.

### 3) DTO-моделі
Додано DTO для кожної сутності:
- Для списку (мінімальний набір полів).
- Для деталей (повний набір полів, потрібний в UI).

### 4) IoC / Dependency Injection
- Репозиторії та сервіси підключаються через **DI** (без створення екземплярів вручну).
- Реєстрація залежностей виконується через IoC контейнер у точці старту застосунку.

## Результат
- Збережено 3 екрани: **список предметів → деталі предмета → деталі заняття**.
- Працює навігація назад.
- Архітектура стала структурованою та масштабованою для наступних лабораторних.
