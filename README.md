# StudyManager — ЛР1

## Структура

- **StudyManager.Storage**  
  Класи для **зберігання даних** (Storage-моделі) та `enum`:
  - `SubjectRecord`, `LessonRecord`
  - `SubjectDomain`, `LessonType`  

- **StudyManager.App**  
  Класи для **створення/відображення/редагування** (View-моделі):
  - `SubjectView` (може містити список занять, має обчислюване поле `TotalDuration`)
  - `LessonView` (має обчислюване поле `Duration`)

- **StudyManager.Services**  
  Сервіс доступу до даних і штучне сховище:
  - `FakeStudyStore` (internal) — містить початкові дані (3 предмети, 12 занять)
  - `StudyRepository` — єдина точка доступу до даних, мапить Storage → App-моделі

- **StudyManager.ConsoleApp**  
  Консольний інтерфейс для перегляду сутностей.

## Логіка роботи застосунку

1. Після запуску консоль отримує список предметів через `StudyRepository.GetAllSubjects()`.
2. Користувач бачить список предметів і може:
   - вибрати предмет за номером
   - оновити список предметів
   - завершити роботу програми
3. Після вибору предмета:
   - відображаються деталі предмета
   - показується список занять для цього предмета
4. Користувач може вибрати заняття та переглянути повну інформацію по ньому.
5. Є можливість повернутися назад і обрати інший предмет або вийти з програми.
